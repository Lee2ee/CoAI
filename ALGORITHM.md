# CoAI 자동매매 알고리즘 문서

## 1. 개요

업비트 KRW 마켓 종목을 주기적으로 스캔하여 **기술 지표 기반 점수 + 스타일별 가중치**를 산출하고,
점수가 기준치를 초과하는 종목에 자동으로 진입·관리·청산하는 시스템입니다.
AI 보조 기능(선택적)이 진입 검증 / 시장 국면 감지 / 손절 분석 / 청산 보조를 담당합니다.

---

## 2. 전체 흐름

```
[스캔 사이클]  APScheduler, trading_style 주기마다
      │
      ▼
scan_market(style, timeframe)
  ├─ 업비트 OHLCV 조회 (최근 150 캔들)
  ├─ 일 거래대금 필터 (스타일별 기준)
  ├─ 기술 지표 점수 계산 (0~100점, 스타일별 가중치)
  └─ 전략 자동 분류 (5종)
      │
      ▼
AI 시장 국면 감지 (15분 캐시, 선택적)
  └─ trending / ranging / volatile → 스타일 / min_score 조정
      │
      ▼ 점수 ≥ min_score
AI 진입 검증 (10분 캐시/종목, 선택적)
  └─ 신뢰도 < 0.6 → 차단
      │
      ▼ 검증 통과
신규 포지션 진입

[실시간 모니터]  asyncio, 0.5초마다
  ├─ 손절(SL) / 익절(TP) 즉시 체결
  └─ 트레일링 스탑 (활성화 후 고점 추적 → 하락 시 청산)

[스캔 사이클 - 포지션 점검]
  ├─ 전략 재평가 (전략 유형 변경 시 SL/TP 재조정)
  ├─ 신호 약화 감지 → 이익 구간 SL 상향 보호
  ├─ 자동 물타기 (avg_down)
  ├─ 자동 추매 (add)
  └─ AI 청산 보조 (5분 캐시/종목, 선택적)
```

## 5. 점수 계산 (0~100점 × 스타일 가중치)

4개 기술 지표를 조합 후 **스타일별 가중치**를 곱해 최종 점수 산출.

### 5-1. 기본 점수 (가중치 적용 전)

#### RSI (최대 30점)

RSI(14) 과매도 구간을 점수화.

| 조건 | 점수 | 신호 |
|------|------|------|
| RSI 20~35 | 10~30점 (30에 가까울수록 고점) | RSI 강한 과매도 |
| RSI 35~45 | 15점 | RSI 과매도 |
| RSI 45~55 | 5점  | 중립 구간 |
| RSI > 55  | 0점  | — |

#### EMA 추세 (최대 20점)

EMA20 / EMA50 배열과 가격 위치.

| 조건 | 점수 | 신호 |
|------|------|------|
| EMA20 > EMA50 (상승 배열) | 12점 | 상승추세 |
| 현재가 > EMA20 | 8점 | 가격 EMA20 상단 |
| EMA20이 EMA50을 상향 돌파 | 별도 전략 분류 | EMA 골든크로스 |

#### MACD (최대 30점)

MACD(12, 26, 9) 교차 및 강세 구간.

| 조건 | 점수 | 신호 |
|------|------|------|
| MACD선 → 시그널선 상향 돌파 | 30점 | MACD 골든크로스 |
| MACD선 > 시그널선 (강세 유지) | 15점 | MACD 강세 구간 |
| MACD선 상승 중 | 5점 | MACD 반등 중 |

#### 거래량 (최대 20점)

직전 20캔들 평균 대비 현재 캔들 거래량 비율.

| 조건 | 점수 | 신호 |
|------|------|------|
| 거래량 3배 이상 | 20점 | 거래량 급증 |
| 거래량 2배 이상 | 12점 | 거래량 증가 |
| 거래량 1.5배 이상 | 6점 | 거래량 소폭 증가 |

### 5-2. 스타일별 가중치 (STYLE_SCORE_WEIGHTS)

같은 지표라도 스타일에 따라 중요도가 다름.

| 스타일 | RSI | EMA | MACD | 거래량 |
|--------|-----|-----|------|--------|
| scalping | 0.4 | 0.5 | **1.6** | **1.5** |
| short    | 1.0 | 1.0 | 1.0  | 1.0 |
| mid      | **1.3** | **1.5** | 0.8 | 0.7 |
| long     | **1.6** | **1.8** | 0.5 | 0.5 |

> scalping은 모멘텀(MACD/거래량) 중심, long은 추세(RSI/EMA) 중심

## 6. 전략 자동 분류

점수 계산 후 신호 조합에 따라 5가지 전략 중 하나로 자동 분류.
**전략 × 스타일** 조합으로 SL/TP가 독립적으로 결정됨.

| 우선순위 | 전략 타입 | 분류 조건 |
|---------|----------|----------|
| 1 | `oversold_bounce` RSI 과매도 반등 | RSI 강한 과매도 (20~35) |
| 2 | `golden_cross` EMA 골든크로스 | EMA20이 EMA50을 금회 상향 돌파 |
| 3 | `volume_breakout` 거래량 돌파 | MACD 골든크로스 + 거래량 3배 이상 |
| 4 | `macd_momentum` MACD 모멘텀 | MACD 골든크로스 |
| 5 | `volume_breakout` 거래량 돌파 | 거래량 3배 이상 + 가격 EMA20 상단 |
| — | `standard` 표준 | 위 조건 미해당 |

### 전략 × 스타일별 SL / TP (%)

| 전략 | scalping | short | mid | long |
|------|----------|-------|-----|------|
| oversold_bounce | SL 1.0 / TP 2.5 | SL 2.5 / TP 7.0 | SL 5.0 / TP 15.0 | SL 10.0 / TP 30.0 |
| golden_cross    | SL 0.8 / TP 2.0 | SL 4.0 / TP 10.0 | SL 6.0 / TP 20.0 | SL 12.0 / TP 36.0 |
| macd_momentum   | SL 0.6 / TP 1.5 | SL 3.0 / TP 9.0 | SL 5.0 / TP 15.0 | SL 9.0 / TP 27.0 |
| volume_breakout | SL 0.7 / TP 2.0 | SL 3.5 / TP 12.0 | SL 5.5 / TP 18.0 | SL 9.0 / TP 30.0 |
| standard        | 글로벌 설정 | 글로벌 설정 | 글로벌 설정 | 글로벌 설정 |

> 글로벌 설정: 봇 설정의 `stop_loss_pct` / `take_profit_pct` 값 사용

## 8. 포지션 관리

### 8-1. 진입

- 잔고의 `position_size_pct`% 만큼 시장가 매수 (PaperBroker)
- 최소 투입금액: 5,000 KRW
- 진입 즉시 전략 SL/TP 가격 계산 + 저장

### 8-2. 실시간 손절/익절 (0.5초 루프)

```
현재가 ≤ stop_loss_price   → 시장가 청산 (stop_loss)
현재가 ≥ take_profit_price → 시장가 청산 (take_profit)
```

### 8-3. 트레일링 스탑

```
조건: pnl_pct ≥ trailing_activate_pct
처리: peak_price 갱신 (현재가 > 기존 고점일 때만)
청산: 현재가 ≤ peak_price × (1 - trailing_pct / 100)
사유: trailing_stop
```

예시 (단타): 4% 이익 달성 → 활성화 → 이후 고점 대비 2% 하락 시 청산

### 8-4. 자동 물타기 (avg_down)

```
조건: 현재가 ≤ 평단가 × (1 - avg_down_threshold_pct / 100)
      AND 물타기 횟수 < max_avg_down (기본 최대 2회)
      AND 잔고 ≥ 5,000 KRW

실행: 초기 포지션 크기의 50% 추가 매수
      → 평단가 재계산 → SL/TP 새 평단 기준 재설정
```

### 8-5. 자동 추매 (add)

```
조건: 현재가 ≥ 평단가 × (1 + add_threshold_pct / 100)
      AND 추매 횟수 < max_add (기본 최대 1회)
      AND 잔고 ≥ 5,000 KRW

실행: 초기 포지션 크기의 25% 추가 매수
```

기본값: 모든 스타일 OFF

### 8-6. 전략 재평가 (스캔 사이클마다)

보유 종목이 스캔 결과에 포함되고 점수 ≥ min_score이면:
- 전략 유형이 변경된 경우 SL/TP를 새 전략 기준으로 재조정

### 8-7. 신호 약화 → SL 보호 상향 (STYLE_SL_PROTECT_PCT)

이익 구간에서 신호 약화 감지 시 SL을 현재가 기준으로 상향.

| 스타일 | SL 상향 폭 |
|--------|-----------|
| scalping | 현재가 대비 -0.3% |
| short    | 현재가 대비 -0.5% |
| mid      | 현재가 대비 -1.0% |
| long     | 현재가 대비 -2.0% |

### 8-8. 청산 사유 종류

| 사유 | 설명 |
|------|------|
| `stop_loss` | 손절가 도달 (실시간 모니터) |
| `take_profit` | 익절가 도달 (실시간 모니터) |
| `trailing_stop` | 트레일링 스탑 발동 (실시간 모니터) |
| `manual` | 사용자 수동 청산 |

## 9. AI 보조 기능

### 9-1. 시장 국면 감지 (detect_regime)

15분마다 1회, BTC 지표를 기반으로 시장 국면 분류.

```
출력: {
  "regime": "trending" | "ranging" | "volatile",
  "style": "scalping" | "short" | "mid" | "long",
  "min_score_delta": -10 ~ +10,
  "reason": "한국어 1문장"
}
```

국면에 따라 봇의 trading_style과 min_score를 동적 조정.

### 9-2. 진입 신뢰도 검증 (check_entry)

종목당 10분 캐시. 진입 후보 발생 시 호출.

```
출력: {
  "confidence": 0.0 ~ 1.0,
  "reason": "한국어 1문장"
}
신뢰도 < 0.6 → 진입 차단 + AI 로그 기록
```

### 9-3. 연속 손절 자기 분석 (analyze_losses)

연속 손절 3회마다 1회 호출 (캐시 없음).

```
출력: {
  "issue": "한국어 1문장",
  "adjusted": ["조정 내용 목록"]
}
```

### 9-4. 청산 타이밍 보조 (check_exit)

수익 중 포지션, 종목당 5분 캐시.

```
출력: {
  "action": "hold" | "close_now" | "tighten_sl",
  "pnl_pct": 현재 수익률,
  "reason": "한국어 1문장"
}
close_now  → 즉시 청산 실행
tighten_sl → SL 현재가 기준 상향 재설정
```

### AI 호출 최소화 전략

- 인메모리 캐시 (`_cache` dict) + TTL 만료 시만 재호출
- 프롬프트 400 토큰 이하로 최소화
- AI 실패 시 폴백: 봇 정상 동작 보장 (기능별 기본값 반환)

## 11. 모의투자 vs 실투자

| 항목 | 모의투자 | 실투자 |
|------|---------|--------|
| 주문 실행 | PaperBroker 가상 체결 | 업비트 실제 API 주문 |
| 시세 데이터 | 실제 업비트 REST API | 동일 |
| 잔고 | 가상 (기본 1,000,000 KRW) | 실제 계좌 잔고 |
| 슬리피지 | 없음 | 실제 호가 스프레드 적용 |
| 수수료 | 가상 0.05% | 실제 업비트 수수료 |

> 현재 봇은 항상 모의투자 모드로 동작 (`is_paper=True` 고정).
> 실투자 전환 시 `PaperBroker` → 업비트 주문 API 연동 필요.

## 12. 알고리즘 고도화 방향

### 단기 (구현 난이도 낮음)
- **동적 종목 스캔**: 고정 25개 → 업비트 전체 KRW 마켓 실시간 거래량 상위 N개 (TODO #6)
- **전략 가중치 자동 조정**: `_strategy_performance` 실적 기반 `STYLE_PREFERRED_STRATEGIES` 동적 변경
- **진입 조건 강화**: 볼린저 밴드 / 스토캐스틱 등 추가 지표 점수에 편입

### 중기 (구현 난이도 중간)
- **전략 빌더 통합**: DB 저장 전략을 봇 스캐너에서 평가 → 개인화 진입 조건 (TODO #5)
- **다중 타임프레임 확인**: 진입 전 상위 타임프레임 추세 확인 (예: 1h 진입 시 4h 방향 일치 여부)
- **포트폴리오 리스크 관리**: 전체 포지션 상관관계 기반 종목별 사이즈 조정

### 장기 (구현 난이도 높음)
- **ML 모델 진입 확률**: 과거 스캔 결과 레이블링 → XGBoost / LightGBM 학습 → AI 검증 대체
- **온라인 학습**: 실시간 청산 결과로 모델 점진적 업데이트
- **강화학습 포지션 사이징**: 시장 국면 + 전략 실적 → 최적 포지션 크기 결정
