## 1. 민감 정보 업로드 금지

* API Key, Secret Key, DB 비밀번호, 토큰 등은 절대 포함하지 않는다.
* `.env`, `application.yml` 등 설정 파일은 `.gitignore`에 추가한다.

예시:

```
.env
application.yml
application.properties
*.key
*.pem
```

---

## 2. 불필요한 파일 제외

* 빌드 결과물: `/build`, `/target`, `/dist`
* IDE 설정: `.idea`, `.vscode`
* OS 파일: `.DS_Store`, `Thumbs.db`

→ 저장소 용량 증가 및 충돌 방지 목적

---

## 3. 커밋 메시지는 명확하게 작성

* 변경 내용을 한눈에 이해할 수 있게 작성한다.
* 형식을 통일한다.

예시:

```
feat: 회원가입 기능 추가
fix: 로그인 오류 수정
refactor: 코드 구조 개선
```

잘못된 예:

```
수정
test
123
```

---

## 4. Push 전에 Pull 먼저 수행

```
git pull origin main
git push origin main
```

→ 최신 코드 반영 후 Push하여 충돌을 방지한다.

---

## 5. 대용량 파일 업로드 금지

* 영상, 데이터셋, 로그 파일 등은 업로드하지 않는다.
* 필요 시 Git LFS 또는 외부 저장소를 사용한다.

---

## 6. 코드 실행 확인 후 Push

* 실행되지 않는 코드, 빌드 실패 상태의 코드는 업로드하지 않는다.
* 최소한 로컬에서 정상 동작 여부를 확인한다.

---

## 7. 코드 정리 및 검토

* 불필요한 주석 및 디버깅 코드를 제거한다.
* 다른 사람이 이해할 수 있는 코드인지 확인한다.

---

## 8.️ 강제 Push 사용 주의

```
git push --force
```

→ 히스토리를 덮어쓰므로 협업 환경에서는 매우 위험하다.

---

## 9. README.md 관리

* 프로젝트 설명, 실행 방법, 기술 스택을 포함한다.
* 항상 최신 상태로 유지한다.
